Instance Variables

  1. Home
  2. Blog Post

Posted on May 25, 2014 at 8:14 PM by Jonathan Young

It took me quite a while to understand what instance variables are; I could be completely wrong even now! This week’s work revolved around classes, and I was forced to learn what certain symbols meant within the world of Ruby classes. $? @? @@?

???

The symbols “$”, “@“, and “@@“ are symbols to differentiate the types of variables you have in your class. $variable_name means that you’re telling Ruby that the variable number after the $ symbol is a global variable and can be accessed literally everywhere in Ruby (global). You can call a $ variable across classes, across methods, everywhere!

@variable_name is what’s called an instance method. To call this variable, you need to define it within a class. After you do that, you can call it anywhere within the class you’ve created - just put the @ sign in front of the variable and you can call it literally anywhere within the Ruby class you’ve written (across methods, in puts/print/return statements), anywhere!

@@variable_name is for class variables. The best way that I’ve heard this explained is that class variables are like global variables for classes. It’s bad to create a bunch of global variables to use within classes because you might take up variable names you need later. You need to create variables within classes. Instance variables (@) start a certain way and are able to be used across the class whenever it’s called, but it’s kind of the blue collar variable; it’s able to move around a lot and get new things saved as new values. You use instance variables when you want a variable that can be changed a lot. You use class variables when you want something fixed and that can be called anywhere within the class.
