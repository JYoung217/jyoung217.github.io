Class Variables

  1. Home
  2. Blog Post

Posted on June 1, 2014 at 8:20 PM by Jonathan Young

I’ve learned quite a bit over the past 6 weeks. Some of it has been easy to digest, and others have not been easy. One of the most difficult things I’ve had to learn so far in my time at DBC includes classes, instances, different types of variables, and when to use them. Today’s blog post will be talking about class variables – what they are, how to use them, and when to use them.

Class variables, are, as the name states, variables that you use within a class. Unlike instance variables, class variables are generally used as global variables within a class. You call the class variable right at the beginning, before you define the initialize method or anything. Also remember that class methods are noted with two @ symbols like so:

class SampleClass

@@ClassVariable = 5

def initialize(param1, param2)

@param1 = param1

@param2 = param2

end

end

Class variables are generally fixed from the get-go and are shared among all instances in the class. It’s also important to note that only one of these variables exists for all objects instantiated in each class. After talking to a few junior developers, I’ve learned that class variables aren’t really used that often and that while Matz (creator of Ruby) created this, it’s seen as bad practice to use classes.

Have you ever used class variables? And if so, why/how?
